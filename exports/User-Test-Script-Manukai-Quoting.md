## Ziele & Forschungsfragen

**Interview-Teil — Quoting-Realität verstehen**
- Wer im Betrieb erstellt heute Angebote? Eine Person oder mehrere?
- Wie viele Angebote pro Woche, und woher kommen sie (Mail mit STEP/PDF, Telefon, Portal)?
- Wer entscheidet über den finalen Preis — und auf welcher Basis (Bauchgefühl, Kalkulation, Excel)?
- Was ist heute das größte Ärgernis am Quoting-Prozess?
- Wie sieht ein „fertiges" Angebot aus, das den Betrieb verlässt?

**Test-Teil — Prototyp-Bewertung**
- Versteht ein Shop-Owner den Stepper-Flow ohne Erklärung?
- Wird die Live-Preisaufschlüsselung (Material / Bearbeitung / Nachbearbeitung / Rüsten / Programmieren / Versand) als nachvollziehbar erlebt?
- Erkennt er, dass Rüst- und Programmierzeit nur einmal pro Feature-Art anfallen, nicht pro Stück?
- Erwartet er, dass Änderungen in **Grundpreise** sofort ins aktive Angebot durchschlagen — und vertraut er diesem Verhalten?
- Wie denkt er über Modell-Feintuning: wann müsste das passieren, wer macht das, wie oft?

## Aufgabe 1: Neues Angebot erstellen — kompletter Stepper (18 Min.)

**Szenario:**
- Ein Kunde hat dir gerade eine STEP-Datei und eine PDF-Zeichnung geschickt.
- Du sollst ihm ein Angebot über 25 Stück aus Edelstahl machen.

**Anweisung:**
- Starte in der Quote-Liste, lade die Dateien hoch, geh den Stepper komplett durch und sag mir am Ende, was das Teil kostet — und warum.

**Subtasks (kurz, ohne Reihenfolge-Zwang):**
- „+ Neues Angebot" finden und öffnen
- Dateien per Drag & Drop oder Picker hinzufügen, „Analysieren" auslösen
- Material wählen, Stückzahl auf 25 setzen
- Form/Maße prüfen
- Features in der Mittelansicht (Relief) überfliegen — Hover und Klick auf eine Feature-Art testen
- Härten oder Trowalisieren aktivieren und beobachten, was im Preis passiert
- Versandart wählen
- Im Preis-Sidebar rechts einen Tooltip öffnen und vorlesen, was du dort verstehst
- Am Ende: nenn den Gesamtpreis und sag, woraus er sich zusammensetzt

| **Flow** |  |  |
| Einstieg | Kommt er ohne Hinweis in den Dialog? |  |

## Aufgabe 2: Grundpreise anpassen (6 Min.)

**Szenario:**
- Dein Edelstahl-Lieferant hat den Kilopreis erhöht.
- Du willst das im System nachziehen und sehen, was mit dem offenen Angebot passiert.

**Anweisung:**
- Geh in den Tab „Grundpreise", änder den Edelstahl-Kilopreis, geh zurück ins Angebot und schau, was passiert ist.

**Follow-ups:**
- Was hast du erwartet, bevor du zurück gegangen bist?
- Welche anderen Werte würdest du in „Grundpreise" auch erwarten?

| Thema | Frage | TP1 |
|-------|-------|-----|
| **Auffindbarkeit** |  |  |
| Tab finden | Findet er „Grundpreise"? |  |
| Richtiges Feld | Kommt er auf Edelstahl / Preis Rund / Preis Flach? |  |
| **Vertrauen** |  |  |
| Erwartung | Was hat er erwartet, dass im Angebot passiert? |  |
| Reaktion | Reagiert er positiv oder verunsichert auf die Live-Aktualisierung? |  |
| Reversibilität | Fragt er nach Speichern / Verwerfen / Defaults? |  |
| **Erwartete fehlende Felder** |  |  |
| Stundenrate | Sucht er sie hier? |  |
| Härten / Trowalisieren | Erwartet er die Raten hier? |  |
| Versandkosten | Erwartet er die hier? |  |
| Geltungsbereich | Pro Angebot oder als Firmen-Default? |  |

## Aufgabe 3: Reflexion — Modell-Feintuning (4 Min., Dialog, kein Klick)

**Szenario-Frage:**
- Stell dir vor, das System schätzt Bearbeitungszeiten falsch — zum Beispiel zu kurz für deine Maschinen.
- Wie würdest du das korrigieren wollen?

**Follow-ups:**
- Wer in deinem Betrieb sollte das dürfen?
- Wie oft erwartest du, das anpassen zu müssen — täglich, monatlich, einmalig?
- Auf welcher Ebene: pro Maschine, pro Material, pro Feature-Art?

| Thema | Frage | TP1 |
|-------|-------|-----|
| **Mental Model** |  |  |
| Korrektur-Idee | Wie stellt er sich die Korrektur vor? |  |

| Audit | Will er sehen, wer was geändert hat? |  |
| **Frequenz & Granularität** |  |  |
