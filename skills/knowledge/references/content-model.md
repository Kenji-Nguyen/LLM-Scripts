# Content Model

## Required Sections in Project Page
1. Project Snapshot
2. Linked Pages
3. Sources
4. Personas
5. Product Requirements
6. Feature Requirements
7. Ideation
8. Research Notes
9. Notes & Decisions

## Section purposes

| Section | Purpose |
|---|---|
| Project Snapshot | Name, goal, stage, stakeholders |
| Linked Pages | External links (trackers, offering docs, Figma) |
| Sources | Provenance log for all inputs (interviews, tests, web, docs) |
| Personas | User segments with needs, tasks, pains, evidence refs |
| Product Requirements | Problem/outcome-level requirements (PRD-###) |
| Feature Requirements | Feature-level specs with acceptance criteria (FRD-###) |
| Ideation | Idea backlog and concepts (IDEA-###) |
| Research Notes | Free-form observations, open questions, competitive notes |
| Notes & Decisions | Timestamped log of decisions, assumptions, and general notes |

## ID conventions
- Sources: `SRC-001`, `SRC-002`, ...
- Product Requirements: `PRD-001`, `PRD-002`, ...
- Feature Requirements: `FRD-001`, `FRD-002`, ...
- Ideas: `IDEA-001`, `IDEA-002`, ...
