# Intake Questions

Ask only what is needed for quality output. Skip questions where context is already clear.

## For any knowledge update
1. Which project should be updated?
2. Which section(s) should change?
3. What new source input should be referenced?
4. Should this be EN, DE, or DE+EN wording?
5. Should we export JSON after update?

## For new project creation
1. What is the project name?
2. What is the project goal (1-2 sentences)?
3. Who are the stakeholders?
4. What language mode? (EN, DE, DE+EN)
5. Any existing documents to migrate?

## For persona updates
1. What research input drives this update? (interview notes, test observations, etc.)
2. Is this a new persona or update to existing?
3. What evidence sources should be referenced?

## For requirement updates
1. What level? (Product requirement = PRD, Feature requirement = FRD)
2. What problem or user story drives this?
3. What evidence supports this?
